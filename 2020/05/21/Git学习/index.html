Git学习

发表于 2020-05-21 更新于 2020-06-26 分类于 Git
本文字数： 2.3k 阅读时长 ≈ 2 分钟
Git本地库的一些操作，包括初始化、基本操作、分支管理等

Git版本控制方式

  • 将工作目录的代码先提交到暂存区，然后再由暂存区提交到本地仓库

Git命令行操作

本地库操作

本地库初始化

  • 进入到需要建立本地仓库的文件夹，执行git init命令，初始化后会生成隐藏文件夹.git

    • 注意：.git文件中存放的是本地库相关的子目录和文件，不要删除，也不要胡乱修改。

基本操作

基础操作

  • 状态查看操作

    • git status
    • 查看工作区、暂存区状态

  • 添加操作

    • git add [file name]
    • 将工作区的“新建/修改”添加到暂存区

  • 提交操作

    • git commit -m "commit message" [file name]
    • 将暂存区的内容提交到本地库

  • 查看历史记录

    • git log

    • 多屏显示控制方式：

      • 空格向下翻页
      • b 向上翻页
      • q 退出

    • git log --pretty=oneline

    • git log --oneline

    • git reflog

      • HEAD@{移动到当前版本需要多少步}

版本前进回退操作

  • 基于索引值操作（最常用、推荐）

    • git reset --hard [局部索引值]
    • git reset --hard f932af0

  • 使用^符号：只能后退

    • git reset --hard HEAD^
    • 注意：一个^表示后退一步，n个表示后退n步

  • 使用~符号：只能后退

    • git reset --hard HEAD~n
    • 注意：表示后退n步

reset命令的三个参数对比

  • --soft参数

    • 仅仅在本地库移动HEDA指针

  • --mixed参数

    • 在本地库移动HEAD指针
    • 重置暂存区

  • --hard参数

    • 在本地库移动HEDA指针
    • 重置暂存区
    • 重置工作区

删除文件并找回

  • 前提：删除前，文件存在时的状态提交到了本地库
  • 操作：git reset --hard [版本索引]

比较文件差异

  • git diff [file name]

    • 将工作区中的文件和暂存区进行比较

  • git diff [版本索引] [file name]

    • 将工作区中的文件和本地库历史记录进行比较

  • git diff 即不带文件名

    • 比较多个文件

分支管理

什么是分支

在版本控制中，使用多条线同时推进多个任务

分支的好处

  • 同时并行推进多个功能开发，提高开发效率
  • 各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支由任何影响。失败的分支删除重新开始即可

分支操作

  • 创建分支

    • git branch [分支名]

  • 查看分支

    • git branch -v

    • 查看远程分支

      • git branch -r

    • 查看所有分支

      • git branch -a

  • 切换分支

    • git checkout [分支名]

  • 将新分支推送到GitHub

    • git push origin [分支名]

  • 删除本地分支

    • git branch -d [分支名]

  • 删除Github远程分支

    • git push origin :[分支名]
    • 分支名前的:代表删除

  • 合并分支

    • 切换到接受修改的分支（被合并，增加新内容）上

      • git checkout [被合并分支名]

    • 执行merge命令

      • git merge [有新内容分支名]

  • 强制提交本地分支覆盖远程分支

    • git push origin [分支名] --force

  • 解决冲突

    • 分支的表现

    • 冲突的解决

      • 编辑文件，删除特殊符号

      • 把文件修改到满意的程度，保存退出

      • git add [文件名]

      • git commit -m "日志信息"

        • 注意：此时commit一定不能带具体文件名

GitHub远程库操作

将本地库与远程库关联

  • 查看远程库地址

    • git remote -v

  • 关联远程库

    • git remote add origin https://github.com/xxx/xxx.git
    • origin即远程库地址的别名，用origin来代表远程库地址

推送操作

  • git push origin [分支名]

克隆操作

  • 命令

    • git clone [远程地址]

  • 效果

    • 完整的把远程库下载到本地
    • 创建origin远程地址别名
    • 初始化本地库

团队成员邀请

  • 项目仓库下settings->Collaborators

  • 搜索需要添加的团队成员

  • 复制邀请链接发送给需要添加的团队成员

  • 需要添加的团队成员通过邀请链接接受邀请

  • 成为团队成员就可正常推送，操作远程库

远程库修改的拉取

  • pull = fetch + merge

  • git fetch [远程库地址别名origin] [远程分支名]

  • git merge [远程库地址别名origin] [远程分支名]

  • 拉取远程库

    • git pull origin [远程分支名]

协同开发冲突的解决

  • 要点

    • 如果不是基于GitHub远程库的最新版所做的修改，不能推送，必须先拉取
    • 拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作即可

跨团队协作

例如：团队B在团队A的基础上开发

团队B

  • 进入团队A的项目仓库

    • fork

    • 本地修改，然后推送到远程

    • pull request

      • Pull request->New pull request->Create pull request(包含日志信息)

    • 开发完成

团队A

  • 进入项目仓库

    • 审核

      • Commits查看提交
      • Files changed 查看文件变化

    • pull request

      • Pull request->点击团队B的pull request

      • 可以comment交流

      • 审核

        • Commits查看提交
        • Files changed 查看文件变化

      • Merge pull request

      • 添加日志信息

      • Confirm merge

      • 合并完成

      • 将远程库修改拉取到本地

配置SSH免密登录

  • 进入家目录

    • cd ~

  • 删除之前的.ssh目录

    • rm -r .ssh/

  • 生成.ssh目录

    • ssh-keygen -t rsa -C [邮箱信息]
    • 默认回车

  • 进入.ssh目录

    • cd .ssh/

  • 将id_rsa.pub文件中的内容复制

  • 在Github上Settings->SSH and GPG keys->New SSH key->将复制内容粘贴->Add SSH key

  • 新建远程库地址别名

    • git remote add origin [仓库SSH的值]

  • 完成推送

  • 不需要输入账户密码，配置完成
